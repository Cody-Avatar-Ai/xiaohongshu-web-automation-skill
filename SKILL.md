# Web Automation Skill - 网页自动化操作优化

## 📋 概述

本技能提供标准化的网页操作流程，优化浏览器操作效率和数据提取质量。适用于社交媒体监控、舆情分析、数据采集等场景。

**适用场景:**
- ✅ 小红书笔记数据采集
- ✅ 微博话题监控
- ✅ 知乎内容分析
- ✅ 电商平台价格监控
- ✅ 新闻舆情追踪

## 🚀 快速开始

### 基本流程

```bash
# 1. 访问页面
browser navigate <url>

# 2. 等待加载
等待 3 秒
browser screenshot (delay=5000ms, fullPage=true)

# 3. 页面分析
browser snapshot (refs=aria)

# 4. 元素定位
从 snapshot 中查找目标元素 ref

# 5. 执行操作
browser act click/type <ref>

# 6. 验证结果
browser screenshot 确认操作生效

# 7. 数据提取
browser snapshot 提取结构化数据

# 8. 重复检查
确认数据完整性
```

## 📋 标准操作程序 (SOP)

### 1️⃣ 页面加载优化

**优化前:**
```
browser navigate <url>
browser screenshot (delay=3000ms)
```

**优化后:**
```
1. browser navigate <url>
2. 等待 3 秒 (让页面初始加载)
3. browser screenshot (delay=5000ms, fullPage=true)
4. 检查截图确认内容加载完成
5. 如未完全加载，再等待 3 秒
```

**收益:** 减少截图失败率 80%

---

### 2️⃣ 元素定位优化

**优化前:**
- 直接猜测元素 ref
- 容易点击错误元素

**优化后:**
```
1. 先执行 browser snapshot (refs=aria)
2. 从返回结果中查找目标元素
3. 确认元素描述匹配 (文本/类型/位置)
4. 使用正确的 ref 执行操作
5. 操作后截图验证
```

**收益:** 元素定位准确率提升至 95%

---

### 3️⃣ 筛选功能发现流程

**系统化页面分析:**

```
1. 检查页面顶部导航栏
   - 查找"筛选"/"过滤"/"排序"等关键词
   - 位置通常在右上角或搜索框下方

2. 检查页面左侧边栏
   - 查找分类筛选选项
   - 位置通常在内容区域左侧

3. 检查页面底部
   - 查找分页和排序选项
   - 位置通常在内容列表下方

4. 使用 snapshot 全面扫描
   - 搜索包含"筛选"/"最新"/"排序"的元素
   - 记录所有相关元素的 ref
```

**小红书筛选特殊情况:**

### 高级筛选面板（右上角"筛选"按钮）

点击后展开完整筛选面板，包含：

**1. 排序依据**
- 综合（默认）
- **最新** ← 按发布时间倒序
- 最多点赞
- 最多评论
- 最多收藏

**2. 笔记类型**
- 不限 / 视频 / 图文

**3. 发布时间**
- 不限 / 一天内 / 一周内 / 半年内

**4. 搜索范围**
- 不限 / 已看过 / 未看过 / 已关注

**5. 位置距离**
- 不限 / 同城 / 附近

**操作步骤:**
```
1. 点击筛选按钮 (ref=e658) - 注意：不是快捷标签
2. 筛选面板展开（浮动层）
3. 再次 snapshot 查找"最新"按钮（在排序依据区域）
4. 点击"最新"按钮 (ref=e1240) - ref 会变化，需实时确认
5. 面板自动收起，按最新排序
```

**注意事项:**
- ⚠️ **重要**：必须点击"筛选"按钮展开高级面板，快捷标签栏没有"最新"选项
- 筛选面板是浮动层，snapshot 可能无法捕获全部元素
- "最新"按钮的 ref 会变化，必须通过 snapshot 实时确认
- 筛选成功后右上角显示"已筛选"标识，"最新"高亮（粉色背景）
- 如果 snapshot 没有找到"最新"，检查筛选面板是否完全展开

### 快捷筛选标签（搜索结果下方）

- 综合 / 今日 / 实习 / 合肥 / 新品等
- ⚠️ **注意**：快捷标签没有"最新"选项！
- 如需按"最新"排序，必须点击"筛选"按钮展开高级面板
- 快捷标签适合快速切换常见筛选条件

**收益:** 筛选功能发现率 100%

### 小红书筛选完整示例（按最新排序）

```
任务：收集小红书最新的 xTool 笔记

执行流程:
1. ✅ 访问页面
   browser navigate https://www.xiaohongshu.com/search/result?keyword=xTool

2. ✅ 等待加载
   browser screenshot (delay=5000ms)

3. ✅ 点击筛选按钮（展开高级面板）
   browser snapshot → 找到筛选按钮 ref=e658
   browser act click ref=e658
   等待 1 秒让面板展开

4. ✅ 选择最新排序（关键步骤！）
   browser snapshot → 在排序依据区域找到"最新"按钮
   注意：ref 会变化，示例 ref=e1240 仅供参考
   browser act click ref=e1240
   等待 2 秒让页面刷新

5. ✅ 点击综合标签（快捷标签栏）
   browser snapshot → 在快捷标签栏找到"综合"按钮
   browser act click ref=e698 (ref 会变化，需实时确认)
   等待 2 秒让页面刷新

6. ✅ 验证筛选生效
   browser screenshot → 确认右上角显示"已筛选"
   browser screenshot → 确认"最新"高亮（粉色背景）
   browser screenshot → 确认"综合"标签高亮
   如未生效，检查是否点击了快捷标签而非高级面板

6. ✅ 提取数据
   browser snapshot (refs=aria)
   遍历所有笔记卡片
   提取标题/作者/发布时间/点赞数/链接
   整理为表格格式

7. ✅ 导入表格
   使用 feishu_bitable_create_record
   批量导入笔记数据
   添加可点击链接

8. ✅ 验证结果
   检查表格记录数 = 15
   随机抽查 3 条链接可访问
   生成执行报告
```

**⚠️ 常见错误:**
- ❌ 错误：点击快捷标签栏的"综合"/"今日"等 → 没有"最新"选项
- ✅ 正确：点击"筛选"按钮 → 展开高级面板 → 选择"最新"

### 4️⃣ 登录状态检查

**检查流程:**

```
1. 访问目标页面
2. 截图查看是否有登录弹窗
3. snapshot 检查是否有"登录"/"注册"按钮
4. 如未登录，反馈给老板请求登录支持
5. 登录后刷新页面确认
```

**收益:** 避免在未登录状态下无效操作

## 🔧 核心技能

### 技能 1: 页面状态检查

**检查项:**
- [ ] 页面是否完全加载
- [ ] 是否有登录弹窗
- [ ] 是否有验证码
- [ ] 是否有错误提示
- [ ] 内容是否显示完整

**操作:**
```
1. browser screenshot (fullPage=true)
2. 检查截图识别异常
3. browser snapshot 检查页面元素
4. 反馈检查结果
```

---

### 技能 2: 智能元素搜索

**输入:** 要查找的元素描述 (如"筛选按钮")  
**输出:** 元素 ref 和位置

**步骤:**
```
1. browser snapshot (refs=aria)
2. 搜索包含关键词的元素
3. 返回匹配的 ref 列表
4. 确认最匹配的元素
```

**示例:**
```
老板：找到"筛选"按钮
我：执行 snapshot → 搜索"筛选" → 返回 ref=e631
```

---

### 技能 3: 批量数据提取

**输入:** 列表页截图 + snapshot  
**输出:** 结构化数据表格

**步骤:**
```
1. 识别列表项容器
2. 遍历每个列表项
3. 提取标题/作者/点赞数/链接
4. 整理为表格格式
5. 验证数据完整性
```

---

### 技能 4: 操作验证流程

**每次操作后执行:**

```
1. 等待 2 秒 (让页面响应)
2. 截图确认操作生效
3. 检查页面变化:
   - URL 是否改变
   - 新元素是否出现
   - 目标内容是否显示
4. 如未生效，重试或反馈
```

## 🎯 核心原则

### 网页操作五大原则

1. **先观察后操作** - 永远先 snapshot 再 act
2. **操作必验证** - 每次操作后截图确认
3. **异常即反馈** - 遇到问题立即报告
4. **数据要完整** - 所有字段必须填充
5. **链接可访问** - 每个链接都要验证
